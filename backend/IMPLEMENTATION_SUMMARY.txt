📈 STATUTE GRAPH STRUCTURE
═══════════════════════════════════════════════════════════════════════════

Each statute is a node in the graph:

{
  "statute_id": {
    "title": "Act Name - Section X: Section Title",
    "related": [
      "statute_that_applies_together_1",
      "statute_that_applies_together_2",
      "legal_aid_statute"
    ],
    "process_flow": ["step1", "step2"],
    "remedy_type": "labour_complaint|protection_order|criminal_complaint|...",
    "next_step": "human-readable action"
  }
}

Example: Domestic Violence
{
  "dv_act_section_3": {
    "title": "DV Act Section 3: Definition of Domestic Violence",
    "related": ["dv_act_section_12", "ipc_section_498a", "ipc_section_376"],
    "process_flow": ["dv_act_section_3", "dv_act_section_12"],
    "remedy_type": "protection_order"
  }
}

When user mentions domestic violence:
  Similarity search finds: dv_act_section_3
  Graph expansion adds: dv_act_section_12, ipc_498a, ipc_376
  User sees: All 4 statutes (complete legal neighborhood)

⚙️ GRAPH EXPANSION ALGORITHM
═══════════════════════════════════════════════════════════════════════════

Simple Python for-loop (just 8 lines!):

def _expand_with_graph(self, seed_ids):
    expanded_ids = set(seed_ids)
    for seed_id in seed_ids:
        if seed_id in self.statute_graph:
            node = self.statute_graph[seed_id]
            expanded_ids.update(node["related"])
    return list(expanded_ids)

That's it. No complex algorithms. Just following dictionaries.

Performance: ~2-5ms for typical expansion
